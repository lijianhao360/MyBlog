Day 01 - Java 语言基础

1. 注释 (Comments)

注释是在程序指定位置添加的说明性信息。

简单理解：就是对代码的一种解释，不参与编译和运行。

2. 关键字 (Keywords)

关键字就是被 Java 赋予了特定涵义的 英文单词。

特点:

  • 全小写：关键字的所有字母均为小写。
  • 高亮显示：常用的代码编辑器（如 IDEA, Eclipse）会对关键字进行特殊的颜色标记。
  • 示例：class (用于创建/定义一个类，类是 Java 最基本的组成单元)。

3. 字面量 (Literals)

字面量告诉程序员数据在程序中的 书写格式。

Day 02 - 数据类型与输入

1. 基本数据类型

概述:
Java 的数据类型分为两大核心类别：基本数据类型（8种）和 引用数据类型（如字符串、数组、类等）。

8种基本数据类型详情:

分类 类型 占用空间 取值范围 适用场景
整数型 byte 1字节 -128 ~ 127 节省内存（如文件流）
short 2字节 -32768 ~ 32767 较少使用
int (默认) 4字节 ±21亿 日常整数使用
long 8字节 非常大 时间戳、金额
浮点型 float 4字节 单精度 对精度要求不高时
double (默认) 8字节 双精度 日常小数使用
字符型 char 2字节 0 ~ 65535 存储单个字符
布尔型 boolean 1字节 true / false 逻辑判断

⚠️ 注意： 定义 long 类型变量时，值末尾需加 L；定义 float 类型时需加 F。

2. 标识符

硬性规则:

  • 由数字、字母、下划线(_)和美元符($)组成。
  • 不能以数字开头。
  • 不能是关键字。
  • 严格区分大小写。

软性建议 (编码规范):

类型 命名法 规范说明 示例
方法、变量 小驼峰 (lowerCamelCase) 首个单词小写，其余单词首字母大写 name, firstName
类名 大驼峰 (UpperCamelCase) 每个单词首字母大写 Student, GoodStudent

Day 03 - 运算符 (Operators)

1. 算术运算符

常用符号: +, -, *, /, % (取余)

1.1 数字相加与类型转换

1. 隐式转换 (自动类型提升)

  • 规则：取值范围小的可以直接转为取值范围大的。
  • 特例：byte, short, char 在运算时，会直接先提升为 int，然后再进行运算。

2. 强制类型转换

  • 规则：取值范围大的赋值给取值范围小的，必须强转。
  • 格式: 目标数据类型 变量名 = (目标数据类型) 被强转的数据;

1.2 字符串相加 (+ 连接符)

  • 当 + 操作中出现字符串时，它是 字符串连接符。
  • 运算顺序：从左到右。
  • "123" + "456" -> "123456"
  • 1 + 99 + "ss" -> "100ss" (先算 1+99=100，再拼接)

1.3 字符相加

字符在参与算术运算时，会使用其对应的 ASCII 码值进行计算。

  • 'a' = 97
  • 'A' = 65
  • '0' = 48

2. 自增自减运算符 (++, --)

  • ++: 变量值加 1
  • --: 变量值减 1

3. 赋值运算符

特点：扩展的赋值运算符 (+=, -=, etc.) 隐含了强制类型转换。

符号 说明 示例 隐含强转
= 赋值 int a=10
+= 加后赋值 a+=b 等价于 a = a+b a = (type)(a+b)
-= 减后赋值 a-=b a = (type)(a-b)

4. 关系运算符

结果一定是 boolean 类型 (true 或 false)。

符号 说明 注意事项
== 判断相等 **千万别写成 =**
!= 判断不等
> 大于
>= 大于等于

5. 逻辑运算符

符号 作用 说明
&& 短路与 左边为 false，右边不执行；结果同 &
` `
! 逻辑非 取反
^ 异或 相同为 false，不同为 true

6. 三元运算符

格式: 关系表达式 ? 表达式1 : 表达式2;

  • 如果关系表达式为 true，结果为表达式 1。
  • 如果关系表达式为 false，结果为表达式 2。

7. 运算符优先级

口诀：单目 > 算术 > 关系 > 逻辑 > 三元 > 赋值

8. 原码、反码、补码

  • 原码: 十进制数据的二进制表现形式，最左边的是符号位，0 为正，1 为负。
  • 反码: 正数的反码是其本身；负数的反码是符号位保持不变，其余位取反。
  • 补码: 正数的补码是其本身；负数的补码是在反码的基础上加 1。

💡 计算机底层运算使用的是补码。

Day 05 - 数组 (Array)

1. 数组介绍

1.1 概念与特点

  • 概念: 数组是一种容器，用来存储同种数据类型的多个值。
  • 特点:
    1. 定长: 数组一旦创建，长度不可改变。
    2. 同构: 容器中的所有元素数据类型必须一致。

1.2 定义格式

格式 写法示例 说明 推荐指数
格式 1 int[] array 数据类型 + [] + 数组名 ⭐⭐⭐⭐⭐ (Java 惯用写法)
格式 2 int array[] 数据类型 + 数组名 + [] ⭐⭐ (C 语言风格，不推荐)

1.3 类型兼容性 (隐式转换)

数组在存储数据时，会发生隐式类型转换，规则与变量赋值一致：

  • int[] 数组：可以存 byte, short, int。
  • double[] 数组：可以存 byte, short, int, long, float, double。
  • 建议: 容器的类型最好和存储的数据类型严格保持一致。

2. 数组初始化

初始化就是在内存中为数组容器开辟空间，并将数据存入容器的过程。

2.1 静态初始化

适用场景: 已经知道数组中具体要存哪些数据。

  • 完整格式: 数据类型[] 数组名 = new 数据类型[]{元素1, 元素2, ...};
  • 简化格式: 数据类型[] 数组名 = {元素1, 元素2, ...};

2.2 动态初始化

适用场景: 只知道数组长度，不知道具体存什么数据。

  • 格式: 数据类型[] 数组名 = new 数据类型[数组长度];

动态初始化默认值规则:

数据类型 默认值
整数 (byte, short, int, long) 0
浮点数 (float, double) 0.0
字符 (char) '\u0000' (空字符)
布尔 (boolean) false
引用类型 (String, 类, 数组等) null

3. 数组的地址值和元素访问

3.1 数组的地址值

当我们直接打印数组变量名时，输出的不是数组的内容，而是数组在内存中的地址哈希值。

  • [ : 表示这是一个数组。
  • I : 表示数组里的类型是 int。
  • @ : 分隔符。
  • 1b6d3586 : 十六进制的内存地址哈希值。

3.2 索引 (Index)

  • 概念: 数组中每个元素对应的编号，也叫下标。
  • 特征: 从 0 开始，连续递增。

3.3 元素访问

  • 格式: 数组名[索引]
  • 赋值: arr[0] = 100;
  • 获取: int num = arr[0];

4. Java 内存分配 (核心)

Java 程序运行在 JVM 中，JVM 对内存进行了划分，其中最常用的是 栈 (Stack) 和 堆 (Heap)。

  1. 栈内存 (Stack): 方法运行时进入使用的内存，变量在这里定义。运行完毕就出栈
  • 例如：int[] arr 这个变量本身在栈中。
  1. 堆内存 (Heap): new 出来的东西（实体/对象）都在堆内存中开辟一个小空间
  • 例如：{10, 20, 30} 这些实际数据在堆中。
  • 堆内存会产生一个地址值，赋值给栈中的变量。
  1. 基本数据类型和引用数据类型

💡 总结: 数组变量保存的是堆内存中的地址，通过地址找到实际的数据。这也就是为什么数组被称为引用数据类型。
