Day 01 - Java 语言基础

1. 注释 (Comments)

注释是在程序指定位置添加的说明性信息。

简单理解：就是对代码的一种解释，不参与编译和运行。

2. 关键字 (Keywords)

关键字就是被 Java 赋予了特定涵义的 英文单词。

特点:

  • 全小写：关键字的所有字母均为小写。
  • 高亮显示：常用的代码编辑器（如 IDEA, Eclipse）会对关键字进行特殊的颜色标记。
  • 示例：class (用于创建/定义一个类，类是 Java 最基本的组成单元)。

3. 字面量 (Literals)

字面量告诉程序员数据在程序中的 书写格式。

4. 变量 (Variables)

变量 就是在程序执行的过程中有可能发生改变的量（数据）。

  • 1. 成员变量（类中、方法外）
    定义位置：在类的内部，所有方法、构造方法、代码块之外。
    分类：
    实例变量（无static）：属于对象，每个对象有独立的一份。
    类变量（有static）：属于类，所有对象共享同一份。
    特点：有默认值（整数 0、浮点 0.0、布尔 false、引用类型 null），无需手动初始化即可使用。

    作用域：整个类都能访问。

  • 2. 局部变量（方法 / 代码块内）
    定义位置：方法、构造方法、for 循环、if 语句、代码块等内部。
    特点：无默认值，必须显式初始化才能使用；作用域仅限于定义它的代码块。
    作用域：出了定义的代码块就失效，无法访问。

Day 02 - 数据类型与输入

1. 基本数据类型

概述:
Java 的数据类型分为两大核心类别：基本数据类型（8种）和 引用数据类型（如字符串、数组、类等）。

8种基本数据类型详情:

分类 类型 占用空间 取值范围 适用场景
整数型 byte 1字节 -128 ~ 127 节省内存（如文件流）
short 2字节 -32768 ~ 32767 较少使用
int (默认) 4字节 ±21亿 日常整数使用
long 8字节 非常大 时间戳、金额
浮点型 float 4字节 单精度 对精度要求不高时
double (默认) 8字节 双精度 日常小数使用
字符型 char 2字节 0 ~ 65535 存储单个字符
布尔型 boolean 1字节 true / false 逻辑判断

⚠️ 注意： 定义 long 类型变量时，值末尾需加 L；定义 float 类型时需加 F。

2. 标识符

硬性规则:

  • 由数字、字母、下划线(_)和美元符($)组成。
  • 不能以数字开头。
  • 不能是关键字。
  • 严格区分大小写。

软性建议 (编码规范):

类型 命名法 规范说明 示例
方法、变量 小驼峰 (lowerCamelCase) 首个单词小写，其余单词首字母大写 name, firstName
类名 大驼峰 (UpperCamelCase) 每个单词首字母大写 Student, GoodStudent

Day 03 - 运算符 (Operators)

1. 算术运算符

常用符号: +, -, *, /, % (取余)

1.1 数字相加与类型转换

1. 隐式转换 (自动类型提升)

  • 规则：取值范围小的可以直接转为取值范围大的。
  • 特例：byte, short, char 在运算时，会直接先提升为 int，然后再进行运算。

2. 强制类型转换

  • 规则：取值范围大的赋值给取值范围小的，必须强转。
  • 格式: 目标数据类型 变量名 = (目标数据类型) 被强转的数据;

1.2 字符串相加 (+ 连接符)

  • 当 + 操作中出现字符串时，它是 字符串连接符。
  • 运算顺序：从左到右。
  • "123" + "456" -> "123456"
  • 1 + 99 + "ss" -> "100ss" (先算 1+99=100，再拼接)

1.3 字符相加

字符在参与算术运算时，会使用其对应的 ASCII 码值进行计算。

  • 'a' = 97
  • 'A' = 65
  • '0' = 48

2. 自增自减运算符 (++, --)

  • ++: 变量值加 1
  • --: 变量值减 1

3. 赋值运算符

特点：扩展的赋值运算符 (+=, -=, etc.) 隐含了强制类型转换。

符号 说明 示例 隐含强转
= 赋值 int a=10
+= 加后赋值 a+=b 等价于 a = a+b a = (type)(a+b)
-= 减后赋值 a-=b a = (type)(a-b)

4. 关系运算符

结果一定是 boolean 类型 (true 或 false)。

符号 说明 注意事项
== 判断相等 **千万别写成 =**
!= 判断不等
> 大于
>= 大于等于

5. 逻辑运算符

符号 作用 说明
&& 短路与 左边为 false，右边不执行；结果同 &
` `
! 逻辑非 取反
^ 异或 相同为 false，不同为 true

6. 三元运算符

格式: 关系表达式 ? 表达式1 : 表达式2;

  • 如果关系表达式为 true，结果为表达式 1。
  • 如果关系表达式为 false，结果为表达式 2。

7. 运算符优先级

在 Java 中，运算符的执行顺序如下（从高到低）：

优先级 描述 运算符
1 括号 ()
2 一元运算符 ++, --, !
3 算术运算符 *, /, % 高于 +, -
4 关系运算符 >, <, == 等
5 逻辑运算符 && 高于 `
6 三元运算符 ? :
7 赋值运算符 = , += 等

口诀：单目 > 算术 > 关系 > 逻辑 > 三元 > 赋值

8. 原码、反码、补码

  • 原码: 十进制数据的二进制表现形式，最左边的是符号位，0 为正，1 为负。
  • 反码: 正数的反码是其本身；负数的反码是符号位保持不变，其余位取反。
  • 补码: 正数的补码是其本身；负数的补码是在反码的基础上加 1。

💡 计算机底层运算使用的是补码。

Day 05 - 数组 (Array)

1. 数组介绍

1.1 概念与特点

  • 概念: 数组是一种容器，用来存储同种数据类型的多个值。
  • 特点:
  1. 定长: 数组一旦创建，长度不可改变。
  2. 同构: 容器中的所有元素数据类型必须一致。

1.2 定义格式

格式 写法示例 说明 推荐指数
格式 1 int[] array 数据类型 + [] + 数组名 ⭐⭐⭐⭐⭐ (Java 惯用写法)
格式 2 int array[] 数据类型 + 数组名 + [] ⭐⭐ (C 语言风格，不推荐)

1.3 类型兼容性 (隐式转换)

数组在存储数据时，会发生隐式类型转换，规则与变量赋值一致：

  • int[] 数组：可以存 byte, short, int。
  • double[] 数组：可以存 byte, short, int, long, float, double。
  • 建议: 容器的类型最好和存储的数据类型严格保持一致。

2. 数组初始化

初始化就是在内存中为数组容器开辟空间，并将数据存入容器的过程。

2.1 静态初始化

适用场景: 已经知道数组中具体要存哪些数据。

  • 完整格式: 数据类型[] 数组名 = new 数据类型[]{元素1, 元素2, ...};
  • 简化格式: 数据类型[] 数组名 = {元素1, 元素2, ...};

2.2 动态初始化

适用场景: 只知道数组长度，不知道具体存什么数据。

  • 格式: 数据类型[] 数组名 = new 数据类型[数组长度];

动态初始化默认值规则:

数据类型 默认值
整数 (byte, short, int, long) 0
浮点数 (float, double) 0.0
字符 (char) '\u0000' (空字符)
布尔 (boolean) false
引用类型 (String, 类, 数组等) null

2.3 动态初始化与静态初始化的区别

3. 数组的地址值和元素访问

3.1 数组的地址值

当我们直接打印数组变量名时，输出的不是数组的内容，而是数组在内存中的地址哈希值。

  • [ : 表示这是一个数组。
  • I : 表示数组里的类型是 int。
  • @ : 分隔符。
  • 1b6d3586 : 十六进制的内存地址哈希值。

3.2 索引 (Index)

  • 概念: 数组中每个元素对应的编号，也叫下标。
  • 特征: 从 0 开始，连续递增。

3.3 元素访问

  • 格式: 数组名[索引]
  • 赋值: arr[0] = 100;
  • 获取: int num = arr[0];

4. Java 内存分配 (核心)

Java 程序运行在 JVM 中，JVM 对内存进行了划分，其中最常用的是 栈 (Stack) 和 堆 (Heap)。

  1. 栈内存 (Stack): 方法运行时进入使用的内存，变量在这里定义。运行完毕就出栈。
  • 例如：int[] arr 这个变量本身在栈中。
  1. 堆内存 (Heap): new 出来的东西（实体/对象）都在堆内存中开辟一个小空间。
  • 例如：{10, 20, 30} 这些实际数据在堆中。
  • 堆内存会产生一个地址值，赋值给栈中的变量。

4.1 基本数据类型和引用数据类型

💡 总结: 数组变量保存的是堆内存中的地址，通过地址找到实际的数据。这也就是为什么数组被称为引用数据类型。

4.2 方法传递内存原理

  1. 方法传递基本数据类型: 传递的是真实的数据值，方法内修改不会影响原变量。
  1. 方法传递引用数据类型: 传递的是地址值，方法内通过地址修改了数据，原变量所指向的数据也会发生改变。

Day 10 - 面向对象进阶 (static 关键字)

static 是 Java 中的一个修饰符，表示“静态的”。它可以用来修饰成员变量和成员方法。被 static 修饰的成员，属于类本身，而不是属于某个具体的对象。

1. static 静态变量

1.1 概念与特点

  • 概念: 被 static 修饰的成员变量，称为静态变量（或类变量）。
  • 特点:
    1. 类加载时产生: 随着类的加载而加载，优先于对象出现。
    2. 全类共享: 该类的所有对象共享这一个变量，在内存中只有一个副本（无论 new 多少个对象，静态变量只有一份）。
  • 调用方式:
    • 类名.变量名 (⭐⭐⭐⭐⭐ 强烈推荐)
    • 对象名.变量名 (不推荐，容易让人误以为是对象特有的)

1.2 代码示例

假设我们有一个 Student 类，同一个班级的学生拥有相同的老师，那么“老师姓名”就可以定义为静态变量。

2. static 静态方法

2.1 概念与特点

  • 概念: 被 static 修饰的成员方法，称为静态方法。
  • 特点: 多用于测试类和工具类中，独立于对象存在。
  • 调用方式:
  • 类名.方法名(参数) (⭐⭐⭐⭐⭐ 强烈推荐)
  • 对象名.方法名(参数) (不推荐)

2.2 核心应用场景：工具类 (Utility Class)

在实际开发中，我们经常需要写一些通用的功能（比如打印数组、获取最大值等）。为了方便调用且节省内存（不需要创建对象），我们会把这些方法定义为静态方法，并把这个类称为工具类。

💡 工具类的设计规范：

  1. 类名要见名知意（例如 ArrayUtils, Math）。
  2. 私有化构造方法：目的就是为了不让外界创建它的对象（因为创建工具类对象没有意义，纯属浪费内存）。
  3. 方法全部使用 static 修饰。

代码示例：定义一个数组工具类

3. static 核心注意事项 (面试高频)

在使用 static 时，必须牢记以下三条铁律：

规则 详细解释 原因剖析
1. 静态方法只能访问静态成员 静态方法中不能直接访问非静态的成员变量和方法。只能访问静态变量和静态方法。 因为静态方法随着类加载就存在了，此时可能还没有创建任何对象。而非静态成员必须依赖对象才能使用。
2. 非静态方法可以访问所有成员 非静态方法中，既能访问非静态的，也能访问静态的。 非静态方法被调用时，必然已经存在了对象，而静态成员早就存在了，所以可以随便访问。
3. 静态方法中不能出现 this 关键字 静态方法中写 this 会直接报错编译失败。 this 代表的是“当前调用该方法的对象”，而静态方法是通过类名直接调用的，压根不需要对象，自然就没有 this。

🌟 巧记口诀：

  • 静态只能访问静态。
  • 非静态可以访问所有。
  • 静态中没有 this。

4. 继承 (Inheritance)

4.1 继承的概念与格式

  • 概念: Java 中提供了一个关键字 extends，用这个关键字，我们可以让一个类和另一个类建立起继承关系。
  • 格式: public class 子类名 extends 父类名 {}
    • 例如: public class Student extends Person {}
  • 称呼: Student 称为子类（派生类），Person 称为父类（基类或者超类）。

4.2 使用继承的好处与时机

  • 好处:
    1. 可以把多个子类中重复的代码抽取到父类中，提高代码的复用性。
    2. 子类可以在父类的基础上，增加其他功能，使子类更加强大。
  • 什么时候用继承: 当类与类之间，存在共性的内容时，并满足子类是父类中的一种（is-a 关系，比如猫是一只动物，学生是一个人），就可以考虑使用继承，来优化代码。

4.3 继承的特点 (核心)

  1. 单继承: Java 只支持单继承，不支持多继承。也就是说，一个子类只能有一个直接父类（不能写成 class A extends B, C）。
  2. 多层继承: Java 支持多层继承。子类 A 继承父类 B，父类 B 可以继承父类 C。此时 C 是 A 的间接父类。
  3. 祖宗类: Java 中所有的类都直接或间接地继承自 Object 类。

4.4 子类能继承父类中的哪些内容？

并不是父类所有的东西都能被子类继承，具体规则如下：

内容 是否能被继承 备注说明
构造方法 ❌ 不能 子类必须有自己的构造方法，不能直接用父类的。
成员变量 ✅ 能 无论私有 (private) 还是非私有，都会被继承到子类中。但私有变量子类不能直接使用，需要通过 getter/setter 访问。
成员方法 ✅ 能 (部分) 只有虚方法表中的方法（非 private、非 static、非 final）才能被继承。

4.5 继承中：成员变量的访问特点

核心原则：就近原则
在子类方法中访问一个变量时，查找顺序如下：

  1. 先在局部位置（当前方法内）找。
  2. 如果没有，去本类成员位置找（使用 this.变量名 可以直接指定）。
  3. 如果没有，去父类成员位置找（使用 super.变量名 可以直接指定）。
  4. 如果父类也没有，就继续往上找，直到 Object 类，要是都没有就报错。

4.6 继承中：成员方法的访问特点

和成员变量一样，调用方法时也遵循就近原则：先在本类中找，本类没有再去父类找。

🌟 核心考点：方法重写 (Override)

  • 应用场景: 当父类的方法不能完全满足子类的需求时，子类可以重写父类的方法。
  • 书写要求:
  1. 子类重写的方法，方法名、参数列表必须与父类完全一致。
  2. 子类重写的方法，访问权限必须大于等于父类（public > protected > 空着不写）。
  3. 子类重写的方法，返回值类型必须小于等于父类。
  • @Override 注解:
    放在重写的方法上面，用来校验该方法是否是有效重写。强烈建议每次重写都加上它！

4.7 继承中：构造方法的访问特点

  • 特点 1: 子类不能继承父类的构造方法。
  • 特点 2: 子类中所有的构造方法默认都会先访问父类中的无参构造方法，再执行自己。

为什么？
因为子类在初始化的时候，可能会用到父类的数据。如果父类没有完成初始化，子类将无法使用这些数据。

底层原理 (super() 机制):

  • 子类构造方法的第一行，默认有一行隐藏的代码：super();，它负责调用父类的无参构造。
  • 如果父类中没有无参构造（只写了带参构造），子类必须在构造方法的第一行，**手动手写 super(参数);** 去调用父类的带参构造，否则直接报错编译不通过。

5. 多态 (Polymorphism)

5.1 什么是多态与前提条件

  • 概念: 多态是指同一种行为（方法），具有多个不同表现形式或形态的能力。简单来说，就是同一个对象，在不同时刻表现出来的不同形态。
  • 形成多态的三个必要前提 (缺一不可):
    1. 有继承/实现关系: 类与类之间必须有继承关系，或者类与接口之间有实现关系。
    2. 有方法重写: 子类必须重写父类的方法（体现出不同的行为）。
    3. 父类引用指向子类对象: 也就是将子类对象赋值给父类类型的变量。例如：Animal a = new Dog();

5.2 多态中成员的访问特点 (🌟 重点/易错点)

当使用多态方式创建对象时（Fu f = new Zi();），访问成员变量和成员方法有着截然不同的规则：

访问目标 查找规则口诀 详细解释 底层原因
成员变量 编译看左边，运行看左边 编译时看父类中有没有这个变量，运行时实际获取的也是父类中变量的值。 变量不存在重写（Override）的概念，声明的是什么类型，就去该类型的空间找。
成员方法 编译看左边，运行看右边 编译时看父类中有没有这个方法，但运行的时候，实际执行的是子类重写后的方法。 方法存在重写，运行时虚拟机会通过对象的实际类型（右边的实体）去虚方法表中查找最终的方法。

5.3 多态的好处与弊端

  • 好处 (极大地提高了扩展性):
    在定义方法时，使用父类类型作为参数，该方法就可以接收这个父类的任意子类对象。体现了代码的通用性和复用性。
  • 弊端:

  • 向上转型: Animal a = new Dog(); (自动转换，子转父)
  • 向下转型: Dog d = (Dog) a; (强制转换，父转子)

⚠️ 转型风险 (ClassCastException):

    • 采用公司域名反写 + 项目名 + 模块名。
    • 必须全部英文小写，见名知意。
    • 示例：com.alibaba.payment，com.baidu.login。

导包 (Import) 的四大注意事项 ⚠️：

  1. 同包原则: 使用同一个包中的类时，不需要导包。
  2. 特权包: 使用 java.lang 包中的类时，不需要导包（如 String, System, Math 等）。
  3. 普通导包: 使用其他包中的类时，必须使用 import 关键字显式导包。
  4. 同名冲突: 如果一个类中需要同时使用两个不同包下同名的类，默认只能导入一个，另一个必须使用全类名（包名 + 类名）来区分。

6.2 final 关键字 (最终的)

final 是 Java 中的一个修饰符，表示“最终的、不可改变的”。它可以用来修饰类、方法和变量。

1. final 修饰的三种情况

修饰目标 特点描述 形象比喻 常见系统类/场景
修饰类 被修饰的类不能被继承。 “太监类” / 绝代单传 String 类、Math 类都是 final 修饰的。
修饰方法 被修饰的方法不能被子类重写 (Override)。 最终解释权归父类所有 核心底层算法，防止被子类恶意篡改。
修饰变量 变量变成常量，只能被赋值一次，不可重新赋值。 铁板钉钉 配合 static 使用定义全局常量。

2. final 修饰变量的本质区别 (🌟 面试高频考点)

当 final 修饰变量时，根据数据类型的不同，表现出的特性有本质区别：

  • 修饰基本数据类型: 变量存储的数据值不能发生改变。
  • 修饰引用数据类型: 变量存储的地址值不能发生改变，但对象内部的属性值是可以改变的！

3. 常量 (Constant)

在实际开发中，final 经常和 static 搭配使用，用来定义全局常量。

  • 命名规范: 所有字母全部大写，多个单词之间使用下划线 _ 隔开。
  • 好处: 见名知意，且只会加载一次，节省内存。

7. 权限修饰符和代码块

7.1 权限修饰符 (Access Modifiers)

  • 概念: 权限修饰符用来控制一个类、或者类中的成员变量、成员方法，在不同的范围内是否能够被访问。
  • 权限修饰符分类 (四大权限从严到宽): private < 空着不写(默认/缺省) < protected < public
权限修饰符 本类中 同包的其他类中 不同包的子类中 不同包的无关类中
private (私有) ✅ ❌ ❌ ❌
默认 (空着不写) ✅ ✅ ❌ ❌
protected (受保护) ✅ ✅ ✅ ❌

💡 实际开发中的使用原则 (极简口诀)：

  1. 成员变量：绝大多数情况下使用 private，隐藏细节。
  2. 成员方法：绝大多数情况下使用 public，方便外界调用。
  3. 特例：如果方法中的代码仅仅是抽取出来供本类其他方法使用的，不希望外界调用，可以使用 private 修饰。

7.2 代码块 (Code Blocks)

在 Java 中，使用 {} 括起来的代码被称为代码块。根据其位置和声明的不同，主要分为三种：

1. 局部代码块

  • 位置: 写在方法内部。
  • 作用: 提前结束变量的生命周期，节约内存。
  • 现状: 在现代 JVM 优化和庞大内存的背景下，局部代码块已经极少使用。

2. 构造代码块 (实例代码块)

  • 位置: 写在类中，方法外。
  • 作用: 把多个构造方法中重复的代码抽取出来。
  • 执行时机: 在每次创建对象（new）时执行，并且优先于构造方法执行。

3. 静态代码块 (🌟 重点/高频使用)

  • 格式: static {}
  • 位置: 写在类中，方法外。
  • 作用: 用于给类进行全局的、一次性的初始化操作（如：加载数据库驱动、初始化静态配置数据等）。
  • 执行时机: 随着类的加载而执行，并且只执行一次！ 绝对优先于对象创建。

💣 面试高频考点：代码块的执行顺序

当一个类中同时存在静态代码块、构造代码块和构造方法时，创建对象时的执行顺序是雷打不动的：
静态代码块 -> 构造代码块 -> 构造方法

8. 抽象类和抽象方法 (Abstract)

8.1 概念与设计思想

  • 什么是抽象方法: 当我们将共性行为抽取到父类中时，由于每一个子类执行的具体内容是不一样的，所以我们在父类中无法编写具体的实现逻辑。这种只有方法声明，没有方法体的方法，就叫抽象方法。
  • 什么是抽象类: 包含抽象方法的类，必须是一个抽象类。
  • 核心思想: 抽象类的本质就是**“制定规则”**。它强制要求所有的子类必须按照规定的格式去实现特定的行为，保证了整个体系的规范性。

8.2 定义格式

  • 抽象方法的定义格式:
    使用 abstract 关键字修饰，并且没有大括号 {}，直接以分号 ; 结尾。

示例: public abstract void eat();

  • 抽象类的定义格式:
    在 class 前面加上 abstract 关键字。

8.3 核心注意事项 (⚠️ 面试/笔试必考)

关于抽象类，必须牢记以下四条铁律：

注意事项 详细解读 原因剖析
1. 不能实例化 抽象类绝对不能使用 new 关键字创建对象。 抽象类中可能包含没有具体实现的抽象方法，如果允许创建对象，调用这些空方法是毫无意义且危险的。
2. 类与方法的关系 含有抽象方法的类一定是抽象类；但抽象类中不一定有抽象方法。 即使一个类没有任何抽象方法，你也可以把它声明为 abstract，目的仅仅是不让外界去实例化它。
3. 构造方法 抽象类中可以有（也建议有）构造方法。 虽然它自己不能创建对象，但它的子类在创建对象时，需要通过 super() 调用父类的构造方法，来初始化继承下来的数据。
4. 子类的硬性要求 抽象类的子类，要么重写抽象类中的所有抽象方法；要么自己也变成一个抽象类。 如果子类没有把所有的“空方法”都实现，那它体内依然有抽象方法，自然就只能当抽象类了。

9. 接口 (Interface)

9.1 接口的概念与定义

  • 概念: 接口是一种比抽象类更加纯粹的“抽象”。它主要用来定义一套行为规范（规则）。只要一个类符合这套规则（实现了这个接口），就可以调用对应的方法。
  • 定义格式: 使用 interface 关键字定义。

9.2 接口与类的关系 (核心)

  • 实现关系: 接口和类之间是实现关系，使用 implements 关键字表示。
  • 不能实例化: 接口和抽象类一样，绝对不能使用 new 关键字直接创建对象。
  • 强制重写: 接口的实现类（子类），必须重写接口中的所有抽象方法；否则，这个实现类本身就必须定义为抽象类。

9.4 各种关系的总结 (突破单继承限制)

Java 的类仅仅支持单继承，但接口的出现完美弥补了这个缺陷：

  1. 类和类的关系: 继承关系，只能单继承，支持多层继承。
  2. 类和接口的关系: 实现关系，可以单实现，也可以多实现。还可以在继承一个类的同时，实现多个接口。
  • 语法: public class 子类 extends 父类 implements 接口A, 接口B {}
  1. 接口和接口的关系: 继承关系，支持多继承！
  • 语法: public interface 接口C extends 接口A, 接口B {}
  • 作用: 如果一个类实现了接口 C，就必须重写 A、B、C 里的所有抽象方法。

9.6 拓展：JDK 8+ 接口的新特性 (🌟 进阶/面试高频)

在 JDK 7 及以前，接口中的方法必须全部是抽象方法。但这就带来了一个痛点：如果项目已经上线，后来发现需要给某个接口增加一个新方法，那么所有实现了该接口的子类都会报错，强制要求重写这个新方法！为了解决“接口升级（向后兼容）”的问题，JDK 8 和 JDK 9 对接口进行了重大增强：

1. JDK 8 新特性：默认方法 (default)

  • 作用: 允许在接口中定义带有方法体的方法。这样在接口升级时，现有的实现类就不需要被迫重写该方法了。
  • 格式: public default 返回值类型 方法名(参数列表) { ... } （public 可以省略）。
  • 使用规则:
    1. 实现类可以继承默认方法，直接通过对象调用。
    2. 实现类也可以重写默认方法（重写时必须去掉 default 关键字）。
    3. 多实现冲突（经典坑点）: 如果一个类实现了多个接口，且这些接口中有同名同参的默认方法，实现类必须强制重写该方法，以解决冲突。

2. JDK 8 新特性：静态方法 (static)

  • 作用: 类似于工具类中的静态方法，提供接口级别的通用功能。
  • 格式: public static 返回值类型 方法名(参数列表) { ... } （public 可以省略）。
  • 使用规则:
    1. 静态方法不能被子类继承，也不能被子类重写。
    2. 只能通过接口名直接调用，不能通过实现类的类名或对象名调用。例如：接口名.静态方法名();

3. JDK 9 新特性：私有方法 (private)

  • 作用: 当接口中的多个 default 默认方法或 static 静态方法中有重复的代码时，我们可以把这些重复代码抽取出来。为了不让外界访问到这些只服务于内部的抽取方法，JDK 9 引入了私有方法。
  • 格式:
    • 普通私有方法 (服务于 default 方法)：private 返回值类型 方法名(参数列表) { ... }
    • 静态私有方法 (服务于 static 方法)：private static 返回值类型 方法名(参数列表) { ... }

10. 内部类 (Inner Class)

10.1 内部类的概念与作用

  • 概念: 在一个类的内部再定义一个完整的类。例如：在 A 类的内部定义了 B 类，那么 B 就是内部类，A 就是外部类。
  • 什么时候使用: 当一个事物的内部，还有一个部分需要一个完整的结构进行描述，而这个内部的完整的结构又只为外部事物提供服务时（比如：汽车类里面的发动机类，人体类里面的心脏类）。
  • 内部类的访问特点:
    1. 内部类可以直接访问外部类的成员，包括私有（private）成员。
    2. 外部类要访问内部类的成员，必须创建内部类的对象。

根据定义的位置和修饰符不同，内部类主要分为四种：成员内部类、静态内部类、局部内部类、匿名内部类。

10.2 成员内部类 (Member Inner Class)

10.4 局部内部类 (Local Inner Class)
