4.1 filter,limit和skip

  • filter 过滤

  • limit 获取前几个元素

  • skip 跳过前几个元素

  • 注意1：中间方法，返回新的Stream流，原来的Stream流只能使用一次，建议使用链式编程

  • 注意2：修改Stream流中的数据，不会影响原来集合或者数组中的数据

  • 完整代码
  • 核心知识点说明
    方法名 类型 作用 示例
    filter(Predicate) 中间操作 过滤元素，仅保留符合条件的元素 filter(s -> s.startsWith("张")) 保留张开头的名字
    limit(long n) 中间操作 截取流的前n个元素，截断后续元素 limit(3) 取集合前3个元素
    skip(long n) 中间操作 跳过流的前n个元素，返回剩余元素 skip(4) 跳过前4个，取后面的元素
    forEach(Consumer) 终端操作 遍历流中元素，执行最终操作 forEach(System.out::println) 打印所有元素
  • 关键注意事项
  1. Stream流的一次性：中间操作会返回新的Stream流，原流只能使用一次，因此必须使用链式编程，不能重复复用同一个流。
  2. 不修改原数据：Stream流的所有操作都是对集合数据的副本进行处理，不会影响原集合/数组的内容。
  3. Lambda简化：filter的匿名内部类写法可以用Lambda表达式大幅简化，代码更简洁、可读性更强。
  4. 分页实现：skip(n).limit(m) 是Stream实现分页的标准写法，比如skip(4).limit(3) 对应「第2页，每页3条」的效果。

5.2 collect

  • collect(Collector collector) 收集流中的数据，放到集合中（List Set Map）

  • 完整代码

  • 核心知识点详解
  • 1. collect() 终端操作

  • 作用：将Stream流中的元素，收集到指定的集合（List/Set/Map）中，是Stream流的终端操作，执行后流关闭，不可复用。

  • 核心工具类：java.util.stream.Collectors，提供了toList()、toSet()、toMap()等静态方法，用于生成对应的收集器。

    1. 三种集合收集方式对比
      收集方式 对应方法 集合特性 适用场景
      List Collectors.toList() 可重复、有序 需要保留元素顺序、允许重复的场景
      Set Collectors.toSet() 不可重复、无序 需要自动去重、不关心顺序的场景
      Map Collectors.toMap() 键唯一、键值对结构 需要将元素转为「键-值」映射的场景
    1. toMap() 关键说明（对应第一张图的注释）

  • 参数一（键生成规则）：Function的两个泛型分别为「流元素类型」和「键的目标类型」，apply方法的入参是流中每个元素，返回值作为Map的键。

  • 参数二（值生成规则）：同理，apply方法的返回值作为Map的值。

  • 键冲突处理：Map键必须唯一，若流中存在重复键（如示例的张无忌），必须添加第三个参数(oldVal, newVal) -> newVal，用新值覆盖旧值解决冲突，否则直接抛异常。

  • 运行结果
  • 注意事项
  1. Map键重复问题：直接使用Collectors.toMap(keyMapper, valueMapper)时，若流中存在重复键，会直接抛出IllegalStateException，必须添加冲突处理参数。
  2. split索引越界：确保元素格式严格符合"姓名-性别-年龄"，否则split("-")数组长度不足，会抛出ArrayIndexOutOfBoundsException。
  3. 类型转换异常：年龄转Integer时，确保字符串是合法数字，否则会抛出NumberFormatException。
  4. 流的一次性：collect是终端操作，同一个流只能执行一次，不能重复复用。

  • 3. 三种写法对比

    写法 特点 适用场景
    匿名内部类 代码冗长，显式实现Function接口，底层逻辑清晰 学习理解、复杂转换逻辑
    Lambda表达式 代码简洁，链式编程，灵活度高 日常开发、简单转换逻辑

  • 三、关键注意事项

  1. 方法引用的前提条件
    • 必须有对应的函数式接口（map依赖Function，toArray依赖IntFunction）

  2. 数组类型匹配要求
    • toArray(String[]::new) 中的数组类型，必须和流中元素类型完全一致（流中是String，数组必须是String[]）
  3. Stream流的一次性
    • map是中间操作，toArray是终端操作，流执行后会关闭，不可重复使用
  4. Student类的规范要求
    • 必须提供getName()方法，否则Student::getName无法引用
    • 构造方法、Getter/Setter是JavaBean规范，必须完整实现
